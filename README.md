# A Hybrid Approach to Intrusion Detection: Autoencoders and Decision Trees for Cybersecurity

## Project Overview

Intrusion Detection Systems (IDS) play a critical role in identifying malicious network activity. However, traditional IDS models often struggle with overfitting, resulting in poor performance when exposed to unseen cyber threats.

This project proposes a hybrid IDS framework that combines **Autoencoder-based feature reduction** with a **Decision Tree classifier** to improve detection accuracy while minimizing overfitting. By reducing unnecessary features and preserving important network traffic patterns, the model achieves strong generalization and reliable intrusion detection.

The results demonstrate that carefully designed autoencoder architectures can significantly improve IDS performance, producing highly accurate and efficient models for cybersecurity applications.

## Project Workflow

### 1. Data Collection

DDoS-related records were extracted from the CSE-CIC-IDS2018 dataset and consolidated into a single dataset for analysis.

### 2. Data Cleaning

Missing values, duplicate records, infinite values, and irrelevant features were removed to improve data quality.

### 3. Data Balancing

Downsampling was applied to create an equal distribution of benign and malicious samples.

### 4. Train-Test Splitting

A stratified 80:20 split was used to preserve class distribution across training and testing datasets.

### 5. Data Normalization

Min-Max Scaling and Z-Score Standardization were applied to ensure consistent feature scaling.

### 6. Feature Reduction

Autoencoders generated compressed feature representations while preserving important intrusion-related patterns.

### 7. Model Training

Encoded features were used to train Decision Tree classifiers across multiple Autoencoder configurations.

### 8. Model Evaluation

Performance was evaluated using Accuracy and RMSE to measure classification quality and overfitting.

### 9. Result Analysis

All Autoencoder configurations were compared to identify the most effective architecture.

---

## Methodology

### Data Preprocessing

Data cleaning, label encoding, feature removal, and handling of missing values ensured high-quality input data.

### Downsampling

The dataset was balanced by reducing the number of benign samples, preventing bias toward the majority class.

### Stratified Sampling

Stratified splitting preserved class distributions and improved evaluation reliability.

### Min-Max Normalization

Features were scaled between 0 and 1 to improve learning stability.

### Z-Score Standardization

Features were standardized around a mean of 0 and a standard deviation of 1.

### Autoencoder-Based Feature Reduction

Autoencoders were used to compress high-dimensional network traffic data into a smaller set of meaningful features while preserving important intrusion-related patterns. This reduced data complexity, improved computational efficiency, and helped minimize overfitting before classification.

### Decision Tree Classification

Decision Trees were trained on encoded features generated by the Autoencoders. They were chosen for their simplicity, interpretability, and strong classification performance on structured data.

### K-Fold Cross Validation

A 5-Fold Cross Validation approach was used to evaluate Autoencoder performance by repeatedly training and validating the model on different subsets of the dataset. This provided a more reliable estimate of performance, reduced evaluation bias, and helped identify overfitting.

### Early Stopping

Early stopping was applied during Autoencoder training to prevent overfitting and improve model generalization.

## Model Architecture

### Autoencoder

| Component            | Description                         |
| -------------------- | ----------------------------------- |
| Input Layer          | Network traffic features            |
| Hidden Layers        | Multiple Dense layers               |
| Bottleneck Layer     | Compressed feature representation   |
| Output Layer         | Reconstruction of original features |
| Activation Functions | ReLU, LeakyReLU, Sigmoid            |
| Optimizer            | Adam                                |
| Loss Function        | Categorical Crossentropy            |

### Decision Tree

The encoded features produced by the Autoencoder were used as input to a Decision Tree classifier. The Decision Tree was chosen because it provides interpretable predictions and performs effectively on reduced feature sets.

### Why the Hybrid Autoencoder–Decision Tree Architecture?

* Autoencoder reduces feature complexity.
* Decision Tree performs efficient classification.
* Feature compression helps reduce overfitting.
* Lower dimensionality reduces computational cost.
* Decision Trees provide interpretable predictions.

Together, they improve detection accuracy while maintaining strong generalization on unseen data.

## Results and Performance

### Evaluation Metrics

The models were evaluated using:

* **Accuracy** – Percentage of correctly classified network traffic records.
* **RMSE (Root Mean Squared Error)** – Measures prediction error. Lower RMSE indicates better performance and lower overfitting.

### Decision Tree Performance on Encoded Data

| Autoencoder | Train Accuracy | Test Accuracy | Train RMSE | Test RMSE |
| ----------- | -------------- | ------------- | ---------- | --------- |
| AE1         | 99.99%         | 99.97%        | 0.0077     | 0.0162    |
| AE2         | 99.89%         | 99.80%        | 0.0337     | 0.0448    |
| AE3         | 87.54%         | 87.44%        | 0.3530     | 0.3544    |
| AE4         | 80.37%         | 80.38%        | 0.4431     | 0.4430    |
| AE5         | 93.96%         | 93.94%        | 0.2458     | 0.2461    |
| AE6         | 88.20%         | 88.08%        | 0.3435     | 0.3453    |

### Interpretation

* **AE1 achieved the best overall performance**, delivering the highest accuracy and lowest RMSE.
* **AE2 also demonstrated strong generalization** with minimal performance degradation.
* **AE3–AE6 showed lower performance**, indicating that increasing model complexity does not always improve results.
* Small differences between training and testing metrics suggest minimal overfitting.
* Lower RMSE values indicate better feature representations and more reliable predictions.

## Conclusion

This project addressed the challenge of balancing accuracy and overfitting in Intrusion Detection Systems using a hybrid Autoencoder–Decision Tree framework. Through preprocessing, balancing, normalization, and feature reduction, the model achieved highly accurate intrusion detection while maintaining strong generalization.

Among all configurations, AE1 emerged as the best-performing model, achieving 99.97% test accuracy and the lowest RMSE. The results demonstrate that combining Autoencoder-based feature reduction with Decision Tree classification can significantly improve IDS performance while minimizing overfitting.
